🌐 Project Overview

❓ Problem Statement

  • Existing tools like Zoom/Meet are heavy & proprietary
  • No lightweight, open-source, self-hostable option
  • Limited customisation in enterprise-grade solutions
  • Expensive licensing for small teams & startups
  • Privacy concerns with centralised video platforms
  • Complex setup process discourages self-hosting

✅ Solution – NexCall

  • Lightweight React SPA with Node.js backend
  • WebRTC for browser-native P2P video calls
  • Open source & fully customisable
  • Self-hostable with free-tier cloud deployment
  • JWT-based secure auth with MongoDB storage
  • Screen sharing, chat — zero plugins needed
  • Responsive — works on any modern browser/device

⭐ Key Features

📹 HD Video & Audio

Crystal-clear WebRTC P2P calls using getUserMedia() with adaptive bitrate. Supports camera toggle and mic mute.

🖥️ Screen Sharing

One-click full-screen sharing via getDisplayMedia(). Smart stream swap using RTCRtpSender.replaceTrack().

💬 Real-time Chat

In-meeting text chat broadcast via Socket.io. Messages appear instantly for all participants — no page refresh.

🔐 Secure Authentication

JWT-based login/register with bcrypt password hashing. Token stored in localStorage; all API routes are protected.

🌙 Modern Dark UI

Glassmorphism cards, Framer Motion animations, Material UI components — themed full dark mode.

📜 Meeting History

Each joined/created meeting is logged with date & room code per user, queryable from the History page.

📡 WebRTC & Real-Time Communication

🔄 Call Setup Flow

1
User joins room via Socket.io join-call event
2
Server broadcasts user-joined to all room members
3
Initiator creates RTCPeerConnection object
4
createOffer() → SDP sent via signalling server
5
Remote peer calls createAnswer() and sends back
6
ICE candidates exchanged through Socket.io server
7
P2P media stream established — video & audio flows directly
8
Chat messages routed via server broadcast to room

⚡ Socket.io Events

  • join-call — user enters a meeting room
  • user-joined — notify existing participants
  • offer — relay SDP offer to remote peer
  • answer — relay SDP answer back
  • ice-candidate — NAT traversal ICE data
  • user-left — cleanup remote video tile
  • chat-message — broadcast to all in room

📐 Key WebRTC APIs Used

  • navigator.mediaDevices.getUserMedia()
  • navigator.mediaDevices.getDisplayMedia()
  • RTCPeerConnection — core P2P object
  • RTCRtpSender.replaceTrack() — screen swap
  • addTrack() / ontrack — stream handling

⚙️ Challenges & Solutions

⚠ ICE Candidate Exchange
PROBLEM
WebRTC ICE negotiation failed on certain NAT types, dropping connections before media could flow.
SOLUTION
Buffered ICE candidates and queued them until remote description was set via setRemoteDescription().
⚠ CORS in Production
PROBLEM
Vercel frontend was blocked by Express backend CORS policy after deployment — 403 errors on all API calls.
SOLUTION
Whitelisted exact Vercel domain in CORS config with credentials: true for JWT cookie passing.
⚠ Screen Share Swap
PROBLEM
Replacing camera track with screen track caused remote peer to see a black screen.
SOLUTION
Used RTCRtpSender.replaceTrack() to hot-swap the video track without full renegotiation.
⚠ JWT Persistence
PROBLEM
Users were logged out on every page refresh — auth state lost from memory.
SOLUTION
Stored JWT in localStorage and rehydrated auth context on app mount using useEffect.
⚠ Duplicate Join Events
PROBLEM
Users in same room received duplicate join events on socket reconnect.
SOLUTION
Tracked connected socket IDs server-side; emit user-joined only for genuinely new socket connections.
